# Осетинский речевой датасет (разговорник Таказова)

Датасет пар **«звуковой фрагмент → текст»** для осетинского языка (иронский
диалект) на основе аудиоразговорника Х. А. Таказова (https://ironau.ru/razgovornik.html).
Каждому осетинскому звуковому фрагменту сопоставлены предложение на осетинском,
перевод на русский и метаданные.

## Что лежит в репозитории

| Путь | Что это |
|------|---------|
| **`dataset.xlsx`** | **Главный файл.** Лист `Лист1` — проверенные на слух пары в формате заказчика. |
| **`sound/`** | Все фрагменты в одной папке: `sound_0000001.mp3 …`, уникальные имена. |
| `raw/` | Исходники разговорника — оригинальные `.mp3` и `.pdf` по темам. |
| `dataset/<тема>/` | Тексты пар (осетинский/русский) по темам — вход для привязки. |
| `tools/editor.html` | Редактор: правка границ по волне, склейка фразы из нескольких кусков, правка текста, отметка «тема готова». |
| `build_edited.py` | Сборка финала из выгрузки редактора (`edited.json`) — только проверенные пары. |

## Как получен датасет

Две стадии: **автоматический черновик** разметки + **ручная проверка на слух**
(итоговый датасет = только проверенное).

### Стадия 1 — авто-черновик (forced alignment)

Главная трудность: осетинский язык **не распознаётся** ASR. Решение —
**forced alignment** (принудительное выравнивание): текст уже известен из PDF,
модель лишь находит, где он звучит.

Конвейер на каждую тему:

1. **Блоки по русским якорям** (`langid_all.py` → `block_align.py`).
   В записи диктор читает блок русских фраз, затем блок их осетинских переводов.
   Русские фразы Whisper распознаёт надёжно — по ним определяется, какие фразы
   PDF лежат в каждом осетинском блоке и в каком порядке.
2. **Forced alignment осетинского** (`auto_align.py`).
   Внутри осетинского блока многоязычная модель **MMS** (Meta, торчаудио
   `MMS_FA`) выравнивает известный осетинский текст (романизованный) к аудио и
   даёт точные границы каждой фразы. Распознавание осетинского не требуется.
3. **Нарезка и сборка** (`build_from_candidates.py`).
   Режет осетинские фразы по границам из оригинала → одна папка `sound/`,
   уникальные имена → собирает `dataset.xlsx` в формате заказчика.

Авто-черновик: из 785 пар ~93% получают границы автоматически. Но **совпадение
осетинского звука с текстом машина проверить не может** (ASR осетинского нет),
поэтому это лишь заготовка.

### Стадия 2 — ручная проверка (`tools/editor.html`)

Человек слушает каждую пару и доводит её: правит границы по волне, при нужде
**склеивает фразу из нескольких несмежных кусков** (напр. «авд» + «сахатыл», где
«сахатыл» звучит один раз), правит текст под звук, лишнее убирает. Тема
отмечается «готова». `build_edited.py` склеивает (ffmpeg) и собирает финал —
**только проверенные пары**. Так датасет растёт «меньше, но качественно».

## Итоговая таблица `dataset.xlsx`

Лист `Лист1`, колонки (как в образце заказчика):

| Колонка | Значение |
|---------|----------|
| `path` | Имя файла + кликабельная ссылка на аудио в репозитории (GitHub) |
| `sentence` | Предложение на осетинском (без скобок, без знаков ударения) |
| `sentence_rus` | Перевод на русский |
| `comment` | `-` или короткая пометка |
| `sentence_domain` | `ose` |
| `gender` | `male` |
| `accents variant locale` | `iron` |
| `duration` | Длительность фрагмента, сек |
| `source` | Из какого PDF разговорника тема |

Ссылки ведут на файлы репозитория, поэтому работают после `git push` при
**публичном** репозитории. Рядом с таблицей лежит папка `sound/` — для
локального воспроизведения.

## Рабочий процесс проверки

1. `gen_editor.py` встраивает авто-черновик границ в `tools/editor.html`.
2. Открываешь `tools/editor.html`, по каждой теме слушаешь и доводишь пары,
   отмечаешь тему «готова», жмёшь «выгрузить результат» → `edited.json`
   (в нём только отмеченные темы, накопительно).
3. `build_edited.py --edited edited.json` склеивает куски и собирает финальный
   `dataset.xlsx` + `sound/` — только проверенное.

## Воспроизвести с нуля
```bash
pip install -r requirements.txt        # + ffmpeg в системе

# Стадия 1 — авто-черновик
python3 langid_all.py                  # langid (faster-whisper)
python3 auto_align.py --all            # MMS forced alignment
python3 gen_editor.py                  # черновик границ -> tools/editor.html

# Стадия 2 — ручная проверка
# открыть tools/editor.html, выверить, выгрузить edited.json
python3 build_edited.py --edited edited.json   # финал из проверенного
```
Стадии 1-разделены: две большие модели (whisper и MMS) не грузятся одновременно
(экономия памяти). Авто-прогоны возобновляемы: готовые темы пропускаются.

## Зависимости
`requirements.txt`: faster-whisper (langid), torch + torchaudio (MMS forced
alignment), pdfplumber, openpyxl, rapidfuzz, pydub. В системе нужен **ffmpeg**.

## Примечание про орфографию
В осетинском тексте сняты знаки ударения, скобочные пометки и варианты из
разговорника убраны. Романизация для выравнивания — приближённая (только для
поиска границ, в датасет не попадает); тексты в таблице — кириллица как в PDF.
