# Осетинский речевой датасет (разговорник Таказова)

Датасет пар **«звуковой фрагмент → текст»** для осетинского языка (иронский
диалект). Источник — аудиоразговорник Таказовой: для каждой темы есть
озвучка (`.mp3`) и текст разворота (`.pdf`). Инструменты режут аудио на
отдельные фразы и собирают одну таблицу, где каждому осетинскому звуковому
фрагменту сопоставлены предложение на осетинском, перевод на русский и
метаданные.

## Что лежит в репозитории

| Путь | Что это |
|------|---------|
| **`dataset.xlsx`** | **Главный файл.** Лист **`Лист1`** — таблица на все 785 пар по формату заказчика. Лист **`проверка`** — строки, которые нужно сверить на слух, с пометками. |
| **`sound/`** | Все звуковые фрагменты в одной папке: `sound_0000001.mp3 … sound_0000785.mp3`. |
| `combined_index.csv` | Карта происхождения каждого фрагмента + авто-флаги качества (аудит). |
| `raw/` | Исходники разговорника — оригинальные `.mp3` и `.pdf` по темам. |
| `dataset/<тема>/` | Промежуточный результат по каждой теме. Из них собран главный файл. |
| `pipeline.py` | Основной инструмент конвейера (4 шага). |
| `build_all.py`, `run_batch.py` | Пакетная обработка папки тем. |
| `build_combined.py` | Сборка единого `dataset.xlsx` + папки `sound/` из потемных результатов. |
| `validate_datasets.py` | Проверка целостности (пустые ячейки, битые символы, пропавшие файлы). |
| `_langid_check.py` | Авто-проверка языка каждого фрагмента (Whisper) — ловит русскую речь, попавшую в осетинские строки. |

## Итоговая таблица `dataset.xlsx`

### Лист `Лист1` — основная таблица (формат заказчика)

Две строки заголовка: человеческая (рус.) и машинная. Колонки:

| Колонка | Значение |
|---------|----------|
| `path` | Имя файла фрагмента + **кликабельная ссылка** на аудио (см. ниже) |
| `sentence` | Предложение на осетинском |
| `sentence_rus` | Перевод на русский |
| `comment` | `-` (короткая пометка; статусы проверки — на листе `проверка`) |
| `sentence_domain` | Язык, `ose` (осетинский) |
| `gender` | Голос диктора, `male` |
| `accents variant locale` | Диалект, `iron` (иронский) |
| `duration` | Длительность фрагмента, сек |
| `source` | Из какого PDF разговорника взята тема |

**Как слушать:** ссылки в колонке `path` ведут на файлы в этом GitHub-репозитории
(`github.com/<owner>/<repo>/blob/main/sound/…`). Клик открывает страницу GitHub
со встроенным плеером. Чтобы ссылки работали, репозиторий нужно **запушить** и
сделать **публичным** (см. ниже). Если предпочитаете локальное воспроизведение —
рядом с `dataset.xlsx` лежит папка `sound/` с теми же файлами.

### Лист `проверка` — что сверить на слух

Сюда вынесены все строки, где привязка звука к тексту может быть неточной
(всего 584). Колонка `что проверить` объясняет причину, ссылка `послушать` —
для быстрой сверки. Приоритетные строки (`⚠`) идут сверху.

## Способ получения датасета

Конвейер (`pipeline.py`), 4 шага на каждую тему:

1. **segment** — режет `.mp3` на отдельные высказывания по паузам (тишине).
2. **langid** — прогоняет клипы через Whisper: определяет язык и расшифровку
   (используется, чтобы отделить русскую речь от осетинской).
3. **pdf** — вытаскивает из PDF пары «осетинский — русский», чистит скобки и
   пометки разговорника (`·`, `°`, знаки ударения, артефакты распознавания),
   приводит к стандартному письменному виду.
4. **align** — «русский якорь»: русские клипы сопоставляются с русским текстом
   из PDF и служат якорями; осетинский фрагмент записи — то, что звучит сразу
   после якоря. Сами русские клипы в датасет не попадают.

Затем `build_all.py` обрабатывает все темы пачкой, а `build_combined.py`
склеивает потемные результаты в один `dataset.xlsx` и одну папку `sound/`
со сквозной нумерацией.

## Проверка пар «звук ↔ текст»

Привязка осетинского звука к тексту делается эвристикой («русский якорь»), и она
**не идеальна**. Каждая подозрительная строка попадает на лист `проверка` с
причиной:

- **`⚠ похоже на русскую речь/диктора`** — Whisper уверенно (≥ 0.90) распознал
  фрагмент как русский. Обычно это объявление названия темы диктором или русская
  реплика, случайно попавшая в осетинскую строку — поправить или удалить.
- **`⚠ очень короткий` / `⚠ длинный фрагмент`** — обрезок одного слова или,
  наоборот, склейка нескольких фраз.
- **`⚠ текст повторяется на нескольких фрагментах`** — один и тот же текст
  привязан к разным клипам (явная ошибка привязки).
- **`групповая привязка — сверить на слух`** — на якорь пришлось несколько
  осетинских клипов, порядок мог сбиться.

Из 785 строк **201 надёжна** (на листе `проверка` их нет), **47** помечены `⚠`
(проверять в первую очередь), остальные — мягкий флаг «сверить на слух».

Надёжной автоматической расшифровки осетинского не существует, поэтому
**окончательная сверка осетинского звука с текстом делается на слух** — для
этого и сделаны кликабельные ссылки. `combined_index.csv` помогает в спорных
случаях: по нему видно, из какой темы и какого исходного клипа собран фрагмент.

## Залить на GitHub

Ссылки на аудио в таблице ведут на этот репозиторий, поэтому работают **после
push** и при **публичном** репозитории:
```bash
git push -u origin main
```
Если репозиторий приватный — ссылки откроются только тем, кто залогинен и имеет
доступ. Локальная папка `sound/` работает в любом случае.

## Установка и воспроизведение

Нужен **ffmpeg**: `brew install ffmpeg` (macOS) / `winget install ffmpeg`
(Windows) / `sudo apt install ffmpeg` (Linux). Затем:
```bash
pip install -r requirements.txt

# 1. потемная обработка исходников из raw/
python3 build_all.py --input raw/ --out dataset/ --model large-v3
# 2. (опц.) авто-проверка языка фрагментов
python3 _langid_check.py
# 3. сборка единого файла и общей папки sound/
python3 build_combined.py
# 4. проверка целостности
python3 validate_datasets.py
```

## Примечание про орфографию
В осетинском тексте сняты знаки ударения, `ў` заменена на `у` (стандартное
письменное написание). Скобочные пометки и варианты из разговорника убраны на
шаге `pdf`. Если нужно сохранять ударения как в книге — поправить `clean_common`
в `pipeline.py` (не удалять `̀`/`́`).
